# AegisHealth AI — Intelligent Health Intake & Wellness Assistant

AegisHealth AI is a production-grade, premium digital health information dashboard designed for medical hackathons. It integrates an adaptive symptom-checker wizard, medication facts lookup, and visual vision analysis for pill packaging/skin condition analysis.

## Key Features

1. **Dashboard-Driven UI**: Vibrant glassmorphic slate layout providing rapid pathways for intake checking, chat messaging, and wellness stats.
2. **Adaptive Symptom Intake**: Begins with a free-text entry, prompts 2 to 5 targeted diagnostic questions dynamically generated by LLM intelligence, and delivers structured JSON health assessments.
3. **Double-Layer Safety Scans**: Client-side (regex) and server-side (regex + LLM) emergency warning triggers for immediate 911 alert banner overlays and crisis lines (for suicidal ideation), bypassing LLM latency.
4. **General Health Chat**: Empathetic Q&A messaging client featuring card-structured symptom summaries when physical complaints are detected.
5. **Medication Facts Lookup**: Sourced profiles with dosage guidelines, Mild vs Severe side effects, and interaction cautions.
6. **Vision-Capable Image Analysis**: Processes uploaded labels, skin irritations, and package boxes, gracefully falling back to simulated mock analyses if Groq vision model catalog rotations fail.

---

## Technical Stack

* **Frontend**: React (Vite), Tailwind CSS, Framer Motion, Lucide React
* **Backend**: FastAPI (Python 3.11+), HTTPX, Pydantic v2
* **LLM Orchestration**: Groq API (OpenAI-compatible) with JSON Mode validation

---

## Environmental Configurations

Create a `.env` file in the `backend/` directory:

```env
GROQ_API_KEY=your_groq_api_key_here
LLM_PROVIDER=groq
GROQ_MODEL=llama-3.3-70b-versatile
GROQ_VISION_MODEL=llama-3.2-11b-vision-preview
PORT=8000
```

*Note: If `GROQ_API_KEY` is not provided, AegisHealth runs in local simulation mode with highly detailed static clinical mock structures.*

## Getting Started

### 1. Launching the Backend

Navigate to the `backend/` directory:

```bash
cd backend
python -m venv venv
# Windows powershell:
.\venv\Scripts\Activate.ps1
# Mac/Linux:
source venv/bin/activate

pip install -r requirements.txt
python -m app.main
```

The API will bind to `http://localhost:8000`.

### 2. Launching the Frontend

Navigate to the `frontend/` directory:

```bash
cd frontend
npm install
npm run dev
```

The web client will mount at `http://localhost:5173`. In local dev, Vite automatically proxy-routes requests to the FastAPI backend port.

## Deployments (Render / Cloud Run)

This project is fully containerized. To build and deploy via Docker:

```bash
docker build -t aegishealth-backend ./backend
docker run -p 8000:8000 --env-file ./backend/.env aegishealth-backend
```
